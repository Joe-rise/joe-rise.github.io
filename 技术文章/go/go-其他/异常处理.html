异常处理

Joe's blog 2023-10-21 18:44:15 异常 go

在Go语言中，panic 和 recover 主要用于处理严重的运行时错误，而 error 和 if 语句则用于处理普通的错误。它们在处理异常和错误方面有不同的应用场景和目的。

# panic 和 recover:

panic 用于表示程序发生了无法恢复的错误，类似于其他语言中的异常。当一个函数内部调用 panic 时，它会停止当前函数的执行，并开始沿着调用栈往上执行，直到遇到相应的 recover。 recover 用于在 defer 函数中捕获 panic，防止程序崩溃，并进行一些清理工作。但是，recover 应该谨慎使用，仅在能够确保程序状态一致性的情况下使用。

func example() {
    defer func() {
        if r := recover(); r != nil {
            fmt.Println("Recovered from panic:", r)
        }
    }()
    // ...
    if somethingBadHappens {
        panic("Something bad happened")
    }
    // ...
}

# error 和 if 判断:

error 类型是Go语言中处理常规错误的标准方式。函数可以返回一个 error 类型，表示可能发生的错误。 使用 if 语句来检查函数是否返回了错误，然后根据情况采取适当的措施。这种方式更符合Go语言的习惯和错误处理模型。

func example() error {
    // ...
    if somethingBadHappens {
        return errors.New("Something bad happened")
    }
    // ...
    return nil
}

推荐的做法：

对于可预测的错误和一般情况下的错误处理，使用 error 和 if 语句更为推荐。这是Go语言中的主流错误处理方式，更符合惯用法，并使代码更易读和可维护。 panic 和 recover 更适用于处理严重错误，例如程序状态不一致、不可修复的错误等。在正常情况下，应避免过度使用 panic 和 recover，以免导致代码难以理解和维护。
